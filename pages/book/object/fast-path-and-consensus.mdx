# Fast Path와 합의

Object Model은 object의 소유권 타입에 따라 가변적인 트랜잭션 실행 경로를 허용합니다.
트랜잭션 실행 경로는 네트워크에서 트랜잭션이 처리되고 검증되는 방법을 결정합니다.
이 섹션에서는 Sui의 다양한 트랜잭션 실행 경로와 합의 메커니즘과의 상호작용을
살펴보겠습니다.

## 동시성 문제

본질적으로 블록체인 기술은 근본적인 동시성 문제에 직면합니다: 탈중앙화 환경에서
여러 당사자가 동시에 동일한 데이터를 수정하거나 접근하려고 할 수 있습니다. 이를 위해서는
네트워크의 일관성을 지원하기 위해 트랜잭션을 순서화하고 검증하는 시스템이 필요합니다. Sui는
합의 메커니즘을 통해 이 문제를 해결하여 모든 노드가 트랜잭션의 순서와 상태에
동의하도록 보장합니다.

Alice와 Bob이 동시에 동일한 자산을 구매하려고 시도하는 마켓플레이스 시나리오를 생각해보세요.
네트워크는 이중 지출을 방지하기 위해 이 충돌을 해결해야 하며, 최대 하나의
트랜잭션만 성공하고 다른 하나는 정당하게 거부되도록 보장해야 합니다.

## Fast Path

그러나 모든 트랜잭션이 동일한 수준의 검증과 합의를 필요로 하는 것은 아닙니다. 예를 들어,
Alice가 자신이 소유한 object를 Bob에게 전송하려는 경우, Alice만이
object에 접근할 권한을 가지므로 네트워크는 이 트랜잭션을 네트워크의 다른 모든 트랜잭션과
관련하여 순서화할 필요 없이 처리할 수 있습니다. 이것을 _fast path_ 실행이라고 하며,
계정 소유 object에 접근하는 트랜잭션은 광범위한 합의 없이 빠르게 처리됩니다.
동시 데이터 접근 없음 -> 더 간단한 문제 -> fast path.

Fast path 실행을 허용하는 또 다른 소유권 모델은 _불변 상태_입니다. 불변
object는 변경할 수 없으므로, 이와 관련된 트랜잭션은 순서화할 필요 없이
빠르게 처리될 수 있습니다.

## 합의 경로

공유 상태에 접근하는 트랜잭션 - Sui에서는 공유 object로 표현됨 - 은
모든 노드에서 상태가 업데이트되고 일관되도록 보장하기 위해 순서화가 필요합니다. 이것을
_합의_를 통한 실행이라고 하며, 공유 object에 접근하는 트랜잭션은 네트워크 일관성을
유지하기 위해 합의 프로세스를 거쳐야 합니다.

## Object가 소유한 Object

마지막으로, 다른 object가 소유한 object는 부모 object와 동일한 규칙을 따른다는 점을
언급하는 것이 중요합니다. 부모 object가 _공유_되면 자식 object도 전이적으로
공유됩니다. 부모 object가 불변이면 자식 object도 불변입니다.

## 요약

- **Fast Path:** 계정 소유 object나 불변 공유 상태와 관련된 트랜잭션은
  광범위한 합의 없이 빠르게 처리됩니다.
- **합의 경로:** 공유 object와 관련된 트랜잭션은 네트워크 무결성을 보장하기 위해
  순서화와 합의가 필요합니다.
- **Object가 소유한 Object:** 자식 object는 부모 object의 소유권 모델을 상속합니다.
