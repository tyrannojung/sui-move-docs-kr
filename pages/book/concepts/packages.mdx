Move는 스마트 컨트랙트를 작성하기 위한 언어입니다 - 블록체인에 저장되고 실행되는 프로그램입니다. 단일 프로그램은 패키지로 구성됩니다. 패키지는 블록체인에 게시되며 [주소](./address)로 식별됩니다. 게시된 패키지는 해당 함수를 호출하는 [트랜잭션](./what-is-a-transaction)을 전송하여 상호작용할 수 있습니다. 또한 다른 패키지의 의존성으로 작동할 수도 있습니다.

> 새 패키지를 생성하려면 `sui move new` 명령어를 사용하세요. 명령어에 대해 더 알아보려면 `sui move new --help`를 실행하세요.

패키지는 module로 구성됩니다 - 함수, 타입 및 기타 항목을 포함하는 별도의 스코프입니다.

```
package 0x...
    module a
        struct A1
        fun hello_world()
    module b
        struct B1
        fun hello_package()
```

## 패키지 구조

로컬에서 패키지는 `Move.toml` 파일과 `sources` 디렉토리가 있는 디렉토리입니다. `Move.toml` 파일 - "패키지 매니페스트"라고 불림 - 은 패키지에 대한 메타데이터를 포함하고, `sources` 디렉토리는 module의 소스 코드를 포함합니다. 패키지는 일반적으로 다음과 같습니다:

```
sources/
    my_module.move
    another_module.move
    ...
tests/
    ...
examples/
    using_my_module.move
Move.toml
```

`tests` 디렉토리는 선택 사항이며 패키지에 대한 테스트를 포함합니다. `tests` 디렉토리에 배치된 코드는 체인에 게시되지 않으며 테스트에서만 사용할 수 있습니다. `examples` 디렉토리는 코드 예제에 사용할 수 있으며, 역시 체인에 게시되지 않습니다.

## 게시된 패키지

개발 중에는 패키지에 주소가 없으며 `0x0`으로 설정해야 합니다. 패키지가 게시되면 module의 바이트코드를 포함하는 블록체인에서 단일 고유 [주소](./address)를 얻습니다. 게시된 패키지는 _불변_이 되며 트랜잭션을 전송하여 상호작용할 수 있습니다.

```
0x...
    my_module: <bytecode>
    another_module: <bytecode>
```
