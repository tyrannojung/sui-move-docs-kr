# Ability: Store

이제 [`key`](./key-ability) ability로 활성화되는 최상위 스토리지 함수들을 이해했으므로,
목록의 마지막 ability인 `store`에 대해 이야기할 수 있습니다.

## 정의

`store`는 타입이 object 내부에 _저장_될 수 있도록 하는 특별한 ability입니다. 이 ability는
타입이 `key` ability를 가진 struct의 필드로 사용되기 위해 필요합니다. 다르게 표현하면,
`store` ability는 값이 object에 _포장_될 수 있도록 합니다.

> `store` ability는 전송 작업에 대한 제한도 완화합니다. 이에 대해서는
> [제한된 전송과 공개 전송](./transfer-restrictions) 섹션에서 더 자세히 다룹니다.

## 예제

이전 섹션에서 이미 `key` ability를 가진 타입들을 사용했습니다: 모든 object는 `UID`
필드를 가져야 하며, 이를 예제에서 사용했습니다; 또한 `Config` struct의 일부로 `Storable` 타입을 사용했습니다.
`Config` 타입도 `store` ability를 가집니다.

```move
/// 이 타입은 `store` ability를 가집니다.
public struct Storable has store {}

/// Config는 `store` ability를 가져야 하는 `Storable` 필드를 포함합니다.
public struct Config has key, store {
    id: UID,
    stores: Storable,
}

/// MegaConfig는 `store` ability를 가진 `Config` 필드를 포함합니다.
public struct MegaConfig has key {
    id: UID,
    config: Config, // 여기 있습니다!
}
```

## `store` Ability를 가진 타입들

Move의 모든 네이티브 타입(참조 제외)은 `store` ability를 가집니다. 여기에는 다음이 포함됩니다:

- [bool](./../move-basics/primitive-types#booleans)
- [부호 없는 정수](./../move-basics/primitive-types#integer-types)
- [vector](./../move-basics/vector)
- [address](./../move-basics/address)

표준 라이브러리에 정의된 모든 타입도 `store` ability를 가집니다. 여기에는 다음이 포함됩니다:

- [Option](./../move-basics/option)
- [String](./../move-basics/string)
- [TypeName](./../move-basics/type-reflection)
