# IDE 설정하기

Move 개발을 위한 가장 인기 있는 두 가지 IDE는 VSCode와 IntelliJ IDEA입니다. 두 IDE 모두 구문 강조와 오류 메시지 같은 기본 기능을 제공하지만, 추가 기능에서는 차이가 있습니다. 어떤 IDE를 선택하든, [Move CLI](./install-sui)를 실행하려면 터미널을 사용해야 합니다.

> **IntelliJ 플러그인은 Move 2024 에디션을 지원하지 않으므로, 일부 구문이 강조 표시되지 않을 수 있습니다.**

## VSCode

- [VSCode](https://code.visualstudio.com/)는 Microsoft의 무료 오픈 소스 IDE입니다.
- [Move (Extension)](https://marketplace.visualstudio.com/items?itemName=mysten.move)는 [Mysten Labs](https://mystenlabs.com)에서 관리하는 Move 언어 서버 확장입니다.
- [Move Formatter](https://marketplace.visualstudio.com/items?itemName=mysten.prettier-move) - [Mysten Labs](https://mystenlabs.com)에서 개발하고 관리하는 Move 코드 포매터입니다.
- [Move Syntax](https://marketplace.visualstudio.com/items?itemName=damirka.move-syntax)는 [Damir Shamanaev](https://github.com/damirka/)가 만든 간단한 Move 구문 강조 확장입니다.

## IntelliJ IDEA

- [IntelliJ IDEA](https://www.jetbrains.com/idea/)는 JetBrains의 상용 IDE입니다.
- [Move Language Plugin](https://plugins.jetbrains.com/plugin/23301-sui-move-language)은 [MoveFuns](https://movefuns.org/)에서 제공하는 IntelliJ IDEA용 Move on Sui 언어 확장입니다.

## Emacs

- [Emacs](https://www.gnu.org/software/emacs/)는 무료 오픈 소스 텍스트 에디터입니다.
- [move-mode](https://github.com/amnn/move-mode)는 [Ashok Menon](https://github.com/amnn)이 만든 Emacs용 Move 모드입니다.

## Zed

- [Zed](https://zed.dev/)는 사람과 AI와의 고성능 협업을 위해 설계된 차세대 코드 에디터입니다.
- [Move](https://github.com/Tzal3x/move-zed-extension)는 [Tzal3x](https://github.com/Tzal3x)가 관리하는 Move 언어 서버 확장입니다.

## Github Codespaces

Github의 웹 기반 IDE는 브라우저에서 바로 실행할 수 있으며 거의 완전한 기능의 VSCode 경험을 제공합니다.

- [Github Codespaces](https://github.com/features/codespaces)
- [Move Syntax](https://marketplace.visualstudio.com/items?itemName=damirka.move-syntax)도 확장 마켓플레이스에서 사용할 수 있습니다.
- [Move Formatter](https://marketplace.visualstudio.com/items?itemName=mysten.prettier-move)도 확장 마켓플레이스에서 사용할 수 있습니다.

## 기타 (CLI)

위에 나열된 도구 중 일부는 CLI 지원 버전이 있습니다.

- [prettier-plugin-move](https://www.npmjs.com/package/@mysten/prettier-plugin-move)는 Prettier@v3 플러그인을 위한 TypeScript 패키지와 터미널에서 실행할 수 있는 바이너리를 포함합니다
