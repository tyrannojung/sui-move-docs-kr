주석은 코드에 메모를 추가하거나 문서화하는 방법입니다. 컴파일러에 의해 무시되며 Move 바이트코드로 변환되지 않습니다. 주석을 사용하여 코드가 하는 일을 설명하거나, 자신이나 다른 개발자를 위한 메모를 추가하거나, 코드의 일부를 임시로 제거하거나, 문서를 생성할 수 있습니다. Move에는 세 가지 유형의 주석이 있습니다: 줄 주석, 블록 주석, 문서 주석.

## 줄 주석

이중 슬래시 `//`를 사용하여 나머지 줄을 주석 처리할 수 있습니다. `//` 이후의 모든 내용은 컴파일러에 의해 무시됩니다.

```move
module book::comments_line;

// 모든 것에 주석을 추가해 봅시다!
fun some_function_with_numbers() {
    let a = 10;
    // let b = 10 이 줄은 주석 처리되어 실행되지 않습니다
    let b = 5; // 여기서 주석은 코드 뒤에 배치됩니다
    a + b; // 결과는 10이 아니라 15입니다!
}
```

## 블록 주석

블록 주석은 코드 블록을 주석 처리하는 데 사용됩니다. `/*`로 시작하고 `*/`로 끝납니다. `/*`와 `*/` 사이의 모든 내용은 컴파일러에 의해 무시됩니다. 블록 주석을 사용하여 한 줄 또는 여러 줄을 주석 처리할 수 있습니다. 심지어 줄의 일부를 주석 처리하는 데도 사용할 수 있습니다.

```move
module book::comments_block;

fun /* 어디에나 주석을 달 수 있습니다 */ go_wild() {
    /* 여기
        저기
        어디에나 */ let a = 10;
    let b = /* 여기도 */ 10; /* 그리고 또 */
    a + b;
}
/* 특정 표현식이나 정의를 제거하는 데 사용할 수 있습니다
fun empty_commented_out() {

}
*/
```

이 예제는 다소 극단적이지만 블록 주석을 사용할 수 있는 모든 방법을 보여줍니다.

## 문서 주석

문서 주석은 코드에 대한 문서를 생성하는 데 사용되는 특별한 주석입니다. 블록 주석과 유사하지만 세 개의 슬래시 `///`로 시작하며 문서화하는 항목의 정의 앞에 배치됩니다.

```move
/// Module has documentation!
module book::comments_doc;

/// This is a 0x0 address constant!
const AN_ADDRESS: address = @0x0;

/// This is a struct!
public struct AStruct {
    /// This is a field of a struct!
    a_field: u8,
}

/// This function does something!
/// And it's documented!
fun do_something() {}
```
