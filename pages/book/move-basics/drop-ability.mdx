# Ability: Drop

`drop` ability는 가장 간단한 ability로, struct의 인스턴스를 _무시_하거나
_폐기_할 수 있도록 허용합니다. 많은 프로그래밍 언어에서 이 동작은 기본으로 간주됩니다. 하지만 Move에서는
`drop` ability가 없는 struct는 무시할 수 없습니다. 이는 모든 자산이 적절히 처리되도록 보장하는 Move
언어의 안전 기능입니다. `drop` ability가 없는 struct를 무시하려고 하면 컴파일 오류가 발생합니다.

```move
module book::drop_ability;

/// 이 struct는 `drop` ability를 가집니다.
public struct IgnoreMe has drop {
    a: u8,
    b: u8,
}

/// 이 struct는 `drop` ability를 가지지 않습니다.
public struct NoDrop {}

#[test]
// `IgnoreMe` struct의 인스턴스를 생성하고 무시합니다.
// 인스턴스를 생성했지만 언패킹할 필요가 없습니다.
fun test_ignore() {
    let no_drop = NoDrop {};
    let _ = IgnoreMe { a: 1, b: 2 }; // 언패킹할 필요 없음

    // 코드가 컴파일되려면 값을 언패킹해야 합니다.
    let NoDrop {} = no_drop; // OK
}
```

`drop` ability는 컬렉션이 더 이상 필요하지 않을 때 특별한 처리가 필요없도록 사용자 정의 컬렉션 타입에 자주 사용됩니다. 예를 들어, `vector` 타입은 `drop`
ability를 가지고 있어서 벡터가 더 이상 필요하지 않을 때 무시할 수 있습니다. 하지만 Move 타입 시스템의 가장 큰 특징은 `drop`을 가지지 않을 수 있다는 것입니다. 이는 자산이
적절히 처리되고 무시되지 않도록 보장합니다.

단일 `drop` ability만 가진 struct는 _Witness_라고 불립니다. _Witness_ 개념은
[Witness and Abstract Implementation](./../programmability/witness-pattern) 섹션에서 설명합니다.

## `drop` Ability를 가진 타입들

Move의 모든 네이티브 타입은 `drop` ability를 가집니다. 여기에는 다음이 포함됩니다:

- [`bool`](./../move-basics/primitive-types#booleans)
- [부호 없는 정수](./primitive-types#integer-types)
- [`vector<T>`](./../move-basics/vector) (`T`가 `drop`을 가질 때)
- [`address`](./../move-basics/address)

표준 라이브러리에 정의된 모든 타입도 `drop` ability를 가집니다. 여기에는 다음이 포함됩니다:

- [`Option<T>`](./../move-basics/option) (`T`가 `drop`을 가질 때)
- [`String`](./../move-basics/string)
- [`TypeName`](./../move-basics/type-reflection)
