# 가시성 수정자

모든 module 멤버는 가시성을 가지고 있습니다. 기본적으로 모든 module 멤버는 _비공개(private)_ 입니다 - 즉, 정의된 module 내에서만 접근할 수 있습니다. 하지만 가시성 수정자를 추가하여 module 멤버를 _공개(public)_ - module 외부에서 볼 수 있게, 또는 _public(package)_ - 같은 패키지 내의 module에서 볼 수 있게, 또는 _entry_ - 트랜잭션에서 호출할 수 있지만 다른 module에서는 호출할 수 없게 만들 수 있습니다.

## 내부 가시성

가시성 수정자가 없는 module에 정의된 함수나 struct는 해당 module에 _비공개_ 입니다. 다른 module에서 호출할 수 없습니다.

```move
module book::internal_visibility;

// 이 함수는 같은 module의 다른 함수에서 호출할 수 있습니다
fun internal() { /* ... */ }

// 같은 module -> internal()을 호출할 수 있습니다
fun call_internal() {
    internal();
}
```

다음 코드는 컴파일되지 않습니다:

```move
module book::try_calling_internal;

use book::internal_visibility;

// 다른 module -> internal()을 호출할 수 없습니다
fun try_calling_internal() {
    internal_visibility::internal();
}
```

struct 필드가 Move에서 볼 수 없다고 해서 그 값이 기밀로 유지되는 것은 아닙니다 &mdash; Move 외부에서 온체인 객체의 내용을 읽는 것은 항상 가능합니다. 암호화되지 않은 비밀을 객체 내부에 저장해서는 안 됩니다.

## 공개 가시성

struct나 함수는 `fun` 또는 `struct` 키워드 앞에 `public` 키워드를 추가하여 _공개_ 로 만들 수 있습니다.

```move
module book::public_visibility;

// 이 함수는 다른 module에서 호출할 수 있습니다
public fun public_fun() { /* ... */ }
```

공개 함수는 다른 module에서 가져와서 호출할 수 있습니다. 다음 코드는 컴파일됩니다:

```move
module book::try_calling_public;

use book::public_visibility;

// 다른 module -> public_fun()을 호출할 수 있습니다
fun try_calling_public() {
    public_visibility::public_fun();
}
```

일부 언어와 달리 struct 필드는 공개로 만들 수 없습니다.

## 패키지 가시성

_패키지_ 가시성을 가진 함수는 같은 패키지 내의 모든 module에서 호출할 수 있지만, 다른 패키지의 module에서는 호출할 수 없습니다. 즉, 패키지에 _내부적_ 입니다.

```move
module book::package_visibility;

public(package) fun package_only() { /* ... */ }
```

패키지 함수는 같은 패키지 내의 모든 module에서 호출할 수 있습니다:

```move
module book::try_calling_package;

use book::package_visibility;

// 같은 패키지 `book` -> package_only()를 호출할 수 있습니다
fun try_calling_package() {
    package_visibility::package_only();
}
```

## 네이티브 함수

[프레임워크](./../programmability/sui-framework)와 [표준 라이브러리](./standard-library)의 일부 함수는 `native` 수정자로 표시됩니다. 이러한 함수는 Move VM에서 기본적으로 제공되며 Move 소스 코드에 본문이 없습니다. native 수정자에 대해 더 알아보려면 [Move Reference](./../../reference/functions#native-functions)를 참조하세요.

```move
module std::type_name;

public native fun get<T>(): TypeName;
```

이것은 `std::type_name`의 예제입니다. 이 module에 대해 더 알아보려면 [리플렉션 챕터](./type-reflection)를 참조하세요.
