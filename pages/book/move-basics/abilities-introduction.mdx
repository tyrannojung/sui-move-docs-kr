# Abilities: 소개

Move는 _타입 abilities_를 커스터마이징할 수 있는 독특한 타입 시스템을 가지고 있습니다.
[이전 섹션](./struct)에서 `struct` 정의와 사용 방법을 소개했습니다.
하지만 `Artist`와 `Record` struct의 인스턴스는 코드가 컴파일되기 위해 언패킹되어야 했습니다. 이것은 _abilities_가 없는 struct의 기본 동작입니다.

> 이 책 전체에서 `Ability: <name>` 형식의 챕터를 보게 될 것입니다. 여기서 `<name>`은 ability의 이름입니다. 이러한 챕터들은 해당 ability를 자세히 다루고, 어떻게 작동하며, Move에서 어떻게 사용하는지 설명합니다.

## Abilities란 무엇인가?

Abilities는 타입에 대해 특정 동작을 허용하는 방법입니다. 이들은 struct 선언의 일부이며 struct 인스턴스에 대해 어떤 동작이 허용되는지 정의합니다.

## Abilities 문법

Abilities는 `has` 키워드 다음에 ability 목록을 사용하여 struct 정의에서 설정됩니다.
abilities는 쉼표로 구분됩니다. Move는 4가지 abilities를 지원합니다: `copy`, `drop`, `key`, 그리고
`store`. 각 ability는 struct 인스턴스에 대한 특정 동작을 정의합니다.

```move
/// 이 struct는 `copy`와 `drop` abilities를 가집니다.
public struct VeryAble has copy, drop {
    // field: Type1,
    // field2: Type2,
    // ...
}
```

## 개요

abilities에 대한 간략한 개요:

> [참조](references)를 제외한 모든 내장 타입은 `copy`, `drop`, `store` abilities를 가집니다. 참조는 `copy`와 `drop`을 가집니다.

- `copy` - struct가 _복사_될 수 있도록 허용합니다. [Ability: Copy](./copy-ability) 챕터에서 설명합니다.
- `drop` - struct가 _폐기_되거나 _버려질_ 수 있도록 허용합니다. [Ability: Drop](./drop-ability) 챕터에서 설명합니다.
- `key` - struct가 스토리지에서 _키_로 사용될 수 있도록 허용합니다. [Ability: Key](./../storage/key-ability) 챕터에서 설명합니다.
- `store` - struct가 _key_ ability를 가진 struct에 _저장_될 수 있도록 허용합니다. [Ability: Store](./../storage/store-ability) 챕터에서 설명합니다.

여기서 간략히 언급하는 것이 중요하지만, 다음 챕터들에서 각 ability에 대해 더 자세히 다루고 사용 방법에 대한 적절한 컨텍스트를 제공할 것입니다.

## Abilities가 없는 경우

abilities가 없는 struct는 폐기, 복사 또는 스토리지에 저장할 수 없습니다. 우리는 이러한 struct를
_Hot Potato_라고 부릅니다. 가볍고 재미있는 이름이지만, abilities가 없는 struct는 뜨거운 감자처럼 - 오직 전달만 할 수 있고 특별한 처리가 필요하다는 것을 기억하는 좋은 방법입니다. Hot
Potato는 Move에서 가장 강력한 패턴 중 하나이며, [Hot Potato Pattern](./../programmability/hot-potato-pattern) 챕터에서 더 자세히 다룹니다.
