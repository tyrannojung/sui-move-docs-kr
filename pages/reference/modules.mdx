# Modules

**Module**은 타입과 해당 타입에 대해 작동하는 함수를 정의하는 핵심 프로그램 단위입니다.
구조체 타입은 Move의 [스토리지](./abilities#key) 스키마를 정의하고, module 함수는
해당 타입의 값과 상호작용하는 규칙을 정의합니다. module 자체도 스토리지에 저장되지만,
Move 프로그램 내에서는 접근할 수 없습니다. 블록체인 환경에서 module은 일반적으로
"publishing"이라고 불리는 과정을 통해 체인에 저장됩니다. 게시된 후에는
[`entry`](./functions#entry-modifier)와 [`public`](./functions#visibility)
함수를 특정 Move 인스턴스의 규칙에 따라 호출할 수 있습니다.

## 구문

module은 다음과 같은 구문을 가집니다:

```text
module <address>::<identifier> {
    (<use> | <type> | <function> | <constant>)*
}
```

여기서 `<address>`는 module의 패키지를 지정하는 유효한 [주소](./primitive-types/address)입니다.

예를 들어:

```move
module 0::test;

public struct Example has copy, drop { i: u64 }

use std::debug;

const ONE: u64 = 1;

public fun print(x: u64) {
    let sum = x + ONE;
    let example = Example { i: sum };
    debug::print(&sum)
}
```

## 이름

`module test_addr::test` 부분은 `test` module이 [패키지 설정](./packages)에서
`test_addr` 이름에 할당된 숫자 [주소](./primitive-types/address) 값으로
게시될 것임을 지정합니다.

module은 일반적으로 (숫자 값을 직접 사용하는 것과는 반대로)
[명명된 주소](./primitive-types/address)를 사용하여 선언해야 합니다. 예를 들어:

```move
module test_addr::test;

public struct Example has copy, drop { a: address }

friend test_addr::another_test;

public fun print() {
    let example = Example { a: @test_addr };
    debug::print(&example)
}
```

이러한 명명된 주소는 일반적으로 [패키지](./packages)의 이름과 일치합니다.

명명된 주소는 소스 언어 수준과 컴파일 중에만 존재하므로, 바이트코드 수준에서는
명명된 주소가 해당 값으로 완전히 대체됩니다. 예를 들어 다음과 같은 코드가 있다면:

```move
fun example() {
    my_addr::m::foo(@my_addr);
}
```

그리고 `my_addr`을 `0xC0FFEE`로 설정하여 컴파일하면, 다음과 동일하게 작동합니다:

```move
fun example() {
    0xC0FFEE::m::foo(@0xC0FFEE);
}
```

소스 수준에서 이 두 가지 다른 접근 방식은 동일하지만, 항상 명명된 주소를 사용하고
해당 주소에 할당된 숫자 값을 사용하지 않는 것이 모범 사례입니다.

module 이름은 `a`부터 `z`까지의 소문자 또는 `A`부터 `Z`까지의 대문자로 시작할 수 있습니다.
첫 번째 문자 이후에는 module 이름에 밑줄 `_`, `a`부터 `z`까지의 문자,
`A`부터 `Z`까지의 문자, 또는 `0`부터 `9`까지의 숫자를 포함할 수 있습니다.

```move
module a::my_module {}
module a::foo_bar_42 {}
```

일반적으로 module 이름은 소문자로 시작합니다. `my_module`이라는 이름의 module은
`my_module.move`라는 이름의 소스 파일에 저장되어야 합니다.

## 멤버

`module` 블록 내의 모든 멤버는 어떤 순서로든 나타날 수 있습니다. 기본적으로 module은
[`타입`](./structs)과 [`함수`](./functions)의 모음입니다. [`use`](./uses) 키워드는
다른 module의 멤버를 참조합니다. [`const`](./constants) 키워드는 module의 함수에서
사용할 수 있는 상수를 정의합니다.

[`friend`](./friends) 구문은 신뢰할 수 있는 module 목록을 지정하기 위한 더 이상 사용되지 않는
개념입니다. 이 개념은 [`public(package)`](./functions#visibility)로 대체되었습니다.
